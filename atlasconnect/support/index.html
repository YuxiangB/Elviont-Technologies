Quick Help

Invitation Access

Enter the invitation code exactly as supplied. Most codes can be redeemed once. After successful validation, access remains on that iPhone.

First Connection

Tap Connect and approve Apple’s request to add the Atlas Connect VPN configuration. iOS must approve the profile before the tunnel can start.

Stuck On Connecting

Use Cancel, confirm Wi-Fi or cellular internet is working, wait a few seconds, and try again. Avoid repeatedly tapping Connect while a request is active.

Unexpected Disconnect

Reconnect after the network becomes stable. A Wi-Fi change, airplane mode, device restart, unreachable server, or iOS resource decision can end a tunnel.

Location Information

The current city and country are approximate and based on the public IP address. VPNs, mobile carriers, and ISP routing can make the displayed location differ from your physical position.

Notifications

If connection alerts do not appear, open iOS Settings, find Atlas Connect under Notifications, and confirm notifications are allowed.

If The VPN Will Not Connect

  1. Confirm the iPhone can load a webpage without the VPN.
  2. Return to Atlas Connect and make sure the previous connection attempt has finished or been cancelled.
  3. Check that an Atlas Connect VPN configuration appears in iOS Settings under General › VPN & Device Management › VPN.
  4. If iOS reports that the saved profile is invalid, remove only the old Atlas Connect profile and approve creation again from the app.
  5. Restart Atlas Connect, then restart the iPhone if the NetworkExtension service remains unresponsive.
  6. Install the latest available Atlas Connect version and try again on a stable network.

Understanding Connection Status

Connecting

iOS has received the request and is starting the Packet Tunnel. Atlas Connect does not report success during this stage.

Connected

The app shows Connected only after iOS confirms that the system VPN tunnel is active.

Reconnecting

The tunnel is reasserting after a temporary network interruption. Allow a short period for iOS to restore the route.

Disconnected

The system tunnel is no longer active and device traffic has returned to the standard network route.

What To Include In A Report

  • iPhone model and iOS version.
  • Atlas Connect version shown on the About page.
  • Whether the device was using Wi-Fi or cellular data.
  • The connection status and complete user-facing error message.
  • The approximate time and time zone when the issue occurred.
  • A screenshot with personal information and public IP details removed.

Privacy And Security

VPN credentials are stored in iOS Keychain. Atlas Connect contains no advertising or third-party analytics SDKs. Connection statistics and preferences are stored locally on the device.
